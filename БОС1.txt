2.Создание указанные в задании учетные записи групп и пользователей
sudo groupadd <nameGroup1>
sudo groupadd <nameGroup1>
sudo useradd -m -G <nameGroup1> <nameUser1>
sudo useradd -m -G <nameGroup1> <nameUser2>
sudo useradd -m -G <nameGroup2> <nameUser2>
sudo useradd -m -G none<nameUser3>
sudo usermod -a -G <nameGroup1> <nameUser1>

3.После этого назначьте права доступа на папки и вложенные файлы, как указано в матрице доступа.
Работа с ACL:
setfacl -m d:u:<nameUser1>:rw <path1>   - права для отдельного пользователя
setfacl -m d:g:<nameGroup1>:rw <path1>   - права для отдельной группы 
setfacl -m d:other::r-- <path1>  - права для всех остальных 

getfacl <path1> - просмотр прав доступа 
d(default) - что-то похожее на настледование прав директории всеми каталогами и файлами внутри неё 
или Назначение применяемого по умолчанию правила (опция -d команды setfacl):
setfacl -dm u::<имя_пользователя>:rwx dirname

4.Останавливаем процесс по его PID: kill <PID>  или kill –KILL <PID>

Если процесс запускается из cron: crontab -e
Удаления (отключения) права выполнения для всех (владельца, группы и остальных пользователей) для указанного файла или каталога в системе файлов:chmod a-x <path>
8)
Команда для запуска сервиса iperf: sudo systemctl start iperf
или iperf -c 2022
Команда для включения автозапуска сервиса iperf:sudo systemctl enable iperf
скопируйте фрагмент журнала с записями, относящимися к этому сервису: sudo systemctl status iperf
Получение фрагмента журнала с записями, относящимися к этому сервису:sudo journalctl -u iperf
Информацию о порте 13 (принадлежность процессу) можно получить с помощью команды:sudo ss -tuln | grep :13
Для получения информации о принадлежности этого процесса сервису можно использовать команду:sudo ss -tulnp | grep :13
Команду для остановки сервиса, использующего порт 13: sudo systemctl stop iperf
Команду для отключения автозапуска сервиса, использующего порт 13:sudo systemctl disable <service_name>
